Steps use checkbox (`- [ ]`) syntax for tracking. (This run: inline execution, autonomous; no git repo → commit steps are verification checkpoints.)

**Goal:** Competition kit for Advanced Tactical RPS: C++ bot SDK + process-isolated match engine + CLI + web UI (human 1v1, watch, match stats, tournament).

**Architecture:** All rules in one header (`sdk/rps.h`) shared by engine, tests, and bots. Bots are separate executables speaking a line protocol with per-move timeouts. `arena` CLI produces pretty text or JSON; Python-stdlib server wraps arena for the vanilla-JS UI. Spec: `docs/superpowers/specs/2026-07-10-rps-arena-design.md`.

**Tech Stack:** C++17 (clang, arm64 mac, POSIX fork/pipe/poll), GNU make, Python 3.12 stdlib, vanilla HTML/CSS/JS.

## Locked interfaces (all tasks must match exactly)

`namespace rps` in `sdk/rps.h`:

```cpp
enum Shape { ROCK=0, PAPER=1, SCISSORS=2 };            // r beats s, s beats p, p beats r
enum Upgrade { NONE=0, PLUS=1, SHIFT=2, BOMB=3 };      // costs 0,1,1,2
const char* shapeName(Shape); char shapeChar(Shape); bool shapeFromChar(char, Shape&);
Shape whatBeats(Shape);   // shape that beats the argument
Shape whatLosesTo(Shape); // shape the argument beats
bool  beats(Shape a, Shape b);
int   upgradeCost(Upgrade);
struct Move {
  Shape base; Upgrade upgrade; Shape disguise;         // disguise meaningful iff SHIFT
  Shape effective() const;                             // disguise if SHIFT else base
  std::string str() const;                             // "r" "p+" "s!" "r?p"
  static bool parse(const std::string&, Move&);
  static Move play(Shape); static Move plus(Shape); static Move bomb(Shape);
  static Move shift(Shape base, Shape as);
};
struct Player { std::array<int,3> hand{{7,7,7}}; int coins=0; bool hasLast=false; Shape lastBase=ROCK; int total() const; };
std::string validateMove(const Player&, const Move&);  // "" if legal, else reason
struct TurnResult { int winner; bool aDestroyed,bDestroyed; int aEarned,bEarned; }; // winner +1/-1/0
TurnResult applyTurn(Player& a, Player& b, const Move& ma, const Move& mb);         // assumes valid
char adjudicateByCards(const Player&, const Player&);  // 'a'|'b'|'d'
struct State {
  int turn=1; Player me, opp; std::vector<std::pair<Move,Move>> history; // (mine,theirs) per past turn
  std::vector<Move> legalMoves() const;
  State next(const Move& mine, const Move& theirs) const;
};
class Bot { public: virtual ~Bot(); virtual std::string name(); virtual Move choose(const State&)=0; };
std::mt19937& rng();                                    // seeded from env RPS_SEED else random_device
int runBot(Bot&);  #define RPS_MAIN(BotClass)           // protocol main loop
```

Protocol (engine→bot / bot→engine), one line each:
- `init player <0|1> time_ms <T> max_turns <N>` → `name <text>`
- `state turn <t> my <r> <p> <s> <coins> opp <r> <p> <s> <coins> mylast <mv|-> opplast <mv|->` → `move <spec>`
- `end <win|lose|draw> reason <text>`

Replay JSON (arena → UI): top `{"a","b","winner":"a|b|draw","reason","forfeit":null|{"side","why"},"turnsPlayed","final":{"ah":[r,p,s],"bh":[..],"ac","bc"},"turns":[...]}`; per turn `{"t","a":{"mv","earned","destroyed"},"b":{...},"win":"a|b|tie","after":{"ah","bh","ac","bc"}}`.
Match JSON: `{"n","aWins","bWins","draws","avgTurns","games":[replay...]}`. Tournament JSON: `{"gamesPerPair","bots":[names],"standings":[{"name","wins","losses","draws","points"}],"grid":[[{"w","l","d"}...]]}`.
Interactive stdout events: `state` (full both-side view), `illegal`, `turn`, `end`; stdin: `move <spec>` / `resign`.

Server API: `GET /api/bots`, `POST /api/build`, `POST /api/play {a,b}`, `POST /api/match {a,b,n}`, `POST /api/tournament {n}`, `POST /api/session {bot,humanSide}`, `POST /api/session/<id>/move {move}`, `DELETE /api/session/<id>`. Static from `web/`.

## Rule decisions under test (from spec §Rule clarifications)

One upgrade max; disguise≠base; pay-then-earn coin order; coin by base shape regardless of outcome, none turn 1; `x+` wins only same effective shape; bomb explodes only on loss; double elimination = draw; turn 100 limit → cards adjudication; bot illegal/timeout/crash = forfeit.
